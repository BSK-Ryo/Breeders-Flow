第1条（目的）

本規程は、____（以下「甲」といいます）が提供する「Breeder Flow」サービス（以下「本サービス」といいます）において、利用者（以下「乙」といいます）の個人情報およびサイト訪問者の情報を適切に取り扱うための基準を定めるものです。

第2条（取得する個人情報）

甲は、本サービスの提供にあたり、以下の個人情報を取得します。

2-1. 乙（ブリーダー）から取得する情報

情報の種類取得目的必須/任意

2-2. サイト訪問者の情報

本サービスで生成されるサイトは静的 HTML であり、サイト訪問者の個人情報を甲のサーバーで収集・保存することはありません。ただし、以下の情報は外部サービスにより取得される場合があります。

  • Google Analytics（GA4）: 乙がトラッキング ID を設定した場合、訪問者のアクセス情報が Google に送信されます。この情報の取り扱いは Google のプライバシーポリシーに従います。
  • LINE 公式アカウント: 訪問者が LINE で問い合わせを行った場合、LINE 社のプライバシーポリシーに従います。

第3条（利用目的）

甲は、取得した個人情報を以下の目的のためにのみ利用します。

  1. 乙のホームページの制作・表示（動物取扱業者表記ページ等）
  2. 乙の LINE 公式アカウントとの連携（メールアドレスによる照合）
  3. サービスに関する連絡・通知
  4. 契約管理・請求事務
  5. サービスの改善・品質向上のための統計分析（個人を特定しない形で）

第4条（個人情報の保管方法）

甲は、乙の個人情報を以下の技術的措置により保護します。

4-1. 暗号化

以下の個人情報は、AES-256-GCM 方式により暗号化した上でデータベースに保存します。

  • 代表者氏名
  • メールアドレス
  • 電話番号
  • 住所
  • 動物取扱業登録番号

暗号化の仕様:

  • 暗号化アルゴリズム: AES-256-GCM（認証付き暗号化）
  • 初期化ベクトル（IV）: 暗号化ごとに 12 バイトのランダム値を生成
  • 暗号鍵: 256 ビット、環境変数として安全に管理

4-2. メールアドレスのハッシュ化

LINE 連携時の照合用として、メールアドレスの SHA-256 ハッシュ値を別途保存します。これにより、暗号化されたメールアドレスを復号することなく照合処理が可能です。

4-3. 管理画面でのマスキング

甲の管理画面において、個人情報は以下のようにマスキング表示されます。

  • メールアドレス: t***@example.com（先頭1文字 + ドメイン）
  • 電話番号: 090-****-5678（先頭3桁 + 末尾4桁）
  • 住所: 東京都...（都道府県名のみ）

4-4. 画像の EXIF 情報削除

乙がアップロードする JPEG 画像から、EXIF メタデータ（GPS 位置情報を含む）を自動的に削除します。これにより、画像に埋め込まれた撮影場所等のプライバシー情報がサイト上で公開されることを防止します。

4-5. インフラストラクチャ

個人情報は以下のクラウドサービス上に保存されます。

データ種別保存先所在地
テキストデータ（暗号化済み）Cloudflare D1（SQLite）Cloudflare グローバルネットワーク
画像ファイル（EXIF 削除済み）Cloudflare R2Cloudflare グローバルネットワーク

第5条（第三者提供）

  1. 甲は、以下の場合を除き、乙の個人情報を第三者に提供しません。
    • 乙の同意がある場合
    • 法令に基づく場合（裁判所の令状、行政機関の法的要請等）
    • 人の生命、身体または財産の保護のために必要がある場合であって、乙の同意を得ることが困難であるとき
  2. 甲は、本サービスの提供に必要な範囲で以下の事業者にデータ処理を委託します。これらは個人情報保護法上の「委託」に該当し、「第三者提供」には該当しません。

第6条（秘密保持義務）

  1. 甲および乙は、本サービスに関連して知り得た相手方の個人情報、営業上の秘密、技術上の秘密その他一切の秘密情報を、厳に秘密として保持し、相手方の事前の書面による承諾なく第三者に開示・漏洩してはなりません。
  2. 甲は、個人情報を取り扱う従業者（業務委託先を含む）に対し、秘密保持義務を課すものとします。
  3. 本条の義務は、本契約終了後 2年間 存続するものとします。

第7条（安全管理措置）

甲は、個人情報の漏洩、滅失、毀損を防止するため、以下の安全管理措置を講じます。

7-1. 組織的安全管理措置

  • 個人情報の取り扱いに関する責任者の設置
  • 個人情報へのアクセス権限の管理
  • 操作ログ（監査ログ）の記録・保存

7-2. 技術的安全管理措置

  • 個人情報の AES-256-GCM 暗号化
  • HTTPS/TLS による通信の暗号化
  • API 認証（トークン方式）によるアクセス制御
  • LINE Webhook の署名検証（HMAC-SHA256）
  • 画像からの EXIF 情報自動削除
  • パスワード・API キーの定時比較（タイミング攻撃防止）

7-3. 物理的安全管理措置

  • クラウドサービス（Cloudflare）のデータセンターセキュリティに準拠

第8条（漏洩時の対応）

  1. 甲は、個人情報の漏洩、滅失、毀損（以下「漏洩等」といいます）が発生し、またはそのおそれがあることを認知した場合、以下の措置を講じます。
    • 乙への速やかな通知（認知後 72 時間以内）
    • 漏洩等の原因の調査
    • 被害の拡大防止措置
    • 再発防止策の策定・実施
    • 個人情報保護委員会への報告（法令上必要な場合）
  2. 甲は、漏洩等の原因、影響範囲、再発防止策について、乙に書面で報告するものとします。

第9条（開示・訂正・削除請求）

  1. 乙は、甲に対し、自己の個人情報の開示を請求することができます。甲は、請求を受けた日から 14日以内 に対応するものとします。
  2. 乙は、自己の個人情報に誤りがある場合、甲に対し訂正を請求することができます。
  3. 乙は、甲に対し、自己の個人情報の削除を請求することができます。ただし、本契約の履行に必要な情報については、契約期間中は削除に応じられない場合があります。
  4. 前各項の請求は、乙本人であることを確認した上で対応するものとします。

第10条（契約終了時のデータ取り扱い）

  1. 本契約の終了後、甲は 30日間 のデータ保持期間を設けます。
  2. データ保持期間中、乙は甲に対し以下のデータのエクスポートを請求することができます。
    • 乙が提供したコンテンツ（テキスト、画像）
    • 動物の登録情報
    • その他乙が登録した情報
  3. データ保持期間の経過後、甲は以下のデータを 完全に削除 します。
    • データベースに保存された暗号化済み個人情報
    • メールアドレスのハッシュ値
    • 画像ストレージ（R2）に保存された全画像ファイル
    • 生成されたサイトファイル一式
    • 操作ログ（監査ログ）
  4. 甲は、削除完了後、乙に通知するものとします。

第11条（監査ログ）

  1. 甲は、個人情報へのアクセスおよび操作について、以下の情報を監査ログとして記録します。
    • 操作日時
    • 操作者の種別（管理者 / システム）
    • 操作内容（閲覧、作成、更新、削除）
    • 対象リソース
    • アクセス元 IP アドレス
  2. 監査ログは、本契約の存続期間中保存し、契約終了後は第10条に従い削除します。

第12条（規程の改定）

  1. 甲は、法令の改正、社会情勢の変化、サービス内容の変更等に応じて、本規程を改定することがあります。
  2. 改定にあたっては、改定内容を乙に通知し、通知日から 30日 を経過した時点で効力を生じるものとします。
